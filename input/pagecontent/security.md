# 安全性與個資保護 (Security & Privacy)

勞工健康檢查紀錄與臨場服務資料包含高度敏感之個人醫療健康個資，其存取、傳輸與儲存必須符合我國《個人資料保護法》及衛福部資訊安全相關規範。

## 1. 資料傳輸與存取控制 (Transmission & Access Control)

*   **傳輸通道安全 (Channel Security)**：
    - 所有 API 交換必須強制使用 **HTTPS (TLS 1.3)** 進行管道加密，防範中間人攻擊。
*   **身分驗證與授權 (AuthN & AuthZ)**：
    - 採用 **OAuth 2.0 (RFC 6749)** 及 **OpenID Connect** 標準進行系統與用戶身分認證。
    - 使用 **智慧醫療憑證** 進行醫師或執行人員之電子簽章確認。
*   **角色存取控制 (RBAC)**：
    - **特約醫護人員**：可存取受檢勞工之詳細病史、理學檢查與檢驗數據。
    - **職安人員/雇主主管**：僅能存取「健康管理分級」、「適性配工建議（工作調整項目）」及「臨場服務發現問題/建議」，不得存取詳細之實驗室檢驗細項值（如血糖、肝功能數值等敏感個資），除非獲得勞工書面個資授權同意。
    - **政府監理機關**：存取通報之去識別化統計報表，或基於職業病調查之特定個案法定資料。

---

## 2. 資料去識別化與最小化原則 (De-identification & Minimization)

*   **最小化原則**：事業單位系統儲存勞工健檢結果時，應採取最小暴露原則。若僅為進行健康分級管理，應僅保留分級標籤與配工內容。
*   **通報去識別化 (Anonymization)**：
    - 當進行群體職業病流行病學統計或上傳至非醫療之行政監理系統時，應去除或雜湊化個人識別資料（如姓名、身分證字號改以雜湊 Token 代替，出生日期僅保留年份與月份）。
*   **稽核追蹤 (Audit Trail)**：
    - 系統應對健檢資料之查詢、新增、修改與刪除動作進行完整日誌紀錄（Audit Log），且稽核軌跡不可被竄改。
